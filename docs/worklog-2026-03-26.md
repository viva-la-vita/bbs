# 工作日志 2026-03-26

## 1. Cloudflare 真实 IP 修复

**问题：** 论坛记录的用户 IP 全部是 `172.20.0.1`（Docker 内网网关），不是用户真实 IP。

**根本原因：** 流量链路为 `Cloudflare → gateway-nginx（80/443）→ bbs-nginx（4000）→ PHP-FPM`。bbs-nginx 看到的连接来源是 Docker 网关，且没有正确解析 `X-Forwarded-For`。

**修复方案：** 修改 `nginx/nginx.conf`，信任 Docker 内网（`172.16.0.0/12`）和 Cloudflare 所有 IP 段，使用 `real_ip_header X-Forwarded-For` + `real_ip_recursive on`，从 `X-Forwarded-For` 链中递归剥去受信任 IP，得到用户真实 IP。

**涉及文件：** `nginx/nginx.conf`

## 2. 用户头像文件丢失与恢复

**问题：** 部分用户头像显示破图（broken image），而非字母头像。

**根本原因：** 之前前端部署时执行了 `rm -rf /var/www/flarum/public/assets/`，该命令将用户上传的头像文件（`avatars/` 子目录）一并删除。

**损失统计：**
- 数据库中有头像记录的用户：5263 人
- 删除后磁盘仅剩：8 个文件
- 从 2025-12-08 备份恢复：4900 个文件（匹配数据库记录 3767 个）
- 无法恢复：约 1496 个（2025-12-08 之后上传/更换的头像）

**恢复命令：**
```bash
tar -xzf ~/backup/bbs-2025-12-08.tar.gz -C / --wildcards '*/assets/avatars/*.png'
docker cp /var/www/flarum/public/assets/avatars/. bbs-flarum-1:/var/www/flarum/public/assets/avatars/
```

**预防措施：** 更新部署文档，禁止使用 `rm -rf /var/www/flarum/public/assets/`，改用以下命令只删编译产物：
```bash
docker exec bbs-flarum-1 find /var/www/flarum/public/assets/ -maxdepth 1 -type f -delete
docker exec bbs-flarum-1 rm -rf /var/www/flarum/public/assets/extensions
```

**注意：** Cloudflare CDN 缓存了部分已删除头像，短期内仍可访问，但随缓存过期会陆续失效。

## 3. 主题贴显示「审核中」问题排查

**问题：** 部分主题贴对普通用户不可见，管理员后台没有「批准主题」按钮。

**排查过程：**
- 检查发现问题主题的 `is_private = 1`，但 `recipients` 表中无收件人记录（正常私信有收件人记录）
- `flags` 表中存在 `type = autoMod`、`reason = 敏感词匹配` 的记录，由 FoF Filter 自动创建

**根本原因：** `bbs-filter` 扩展修复了 FoF Filter 的正则表达式 bug（之前过滤器因 regex 错误完全失效），修复后过滤器开始正常工作，匹配到敏感词的帖子被自动设为 `is_private = 1`。

**数据统计（534 条 is_private=1 且无收件人的主题）：**
- 管理员手动删除（hidden_at 非空）：77 条 → 保持隐藏
- 过滤器拦截（is_approved=0）：456 条 → 需在后台 Flags 页面逐一审核
- 疑似错误隐藏（is_approved=1）：1 条 → 已手动修复

**修复主题 32974（炮机评测）：**
```bash
UPDATE discussions SET is_private = 0 WHERE id = 32974;
```

**正确工作流：** 后台 → Flags 页面 → 驳回举报（恢复合法帖）或删除（垃圾帖）
